一对多(多对一)实现方式：在多的一方建立外键，指向一的一方的主键；

多对多实现方式：建立第三张中间表，中间表至少包含两个外键，分别关联两方主键；

一对一实现方式：在任意一方建立外键，关联对方主键，并设置外键唯一；

子查询：查询中嵌套查询，称嵌套的查询为子查询；

事务：包含一组数据库操作命令的一种机制、一个操作序列；

事务把所有的命令作为一个整体一起向系统提交或撤销操作请求，这一组数据库命令要么同时成功，要么同时失败；

事务是一个不可分割的工作逻辑单元；

开启事务：START TRANSACTION 或者BEGIN

提交事务：COMMIT (提交事务之后才正式在内存中执行事务操作所涉及的更改，不提交修改不会生效)

回滚事务：ROLLBACK (事务执行过程出现异常时，回滚至事务开启之前的状态)

为什么使用向下转型？

有了对象的多态性以后，内存中实际上是加载了子类特有的属性和方法，但是由于变量声明为父类类型，导致编译时，只能调用父类中声明的属性和方法。子类特有的属性和方法不能调用。如何才能调用子类特有的属性和方法? 使用向下转型->强制类型转换符 （） 。

注：(1)使用强转时，可能会出现ClassCastException的异常,避免方法，在向下转型前先进行instanceof的判断，返回true时才进行向下转型；

a instanceof A:判断对象a是否是类A的实例；是则返回true；要求a所属的类与类A必须是子类和父类的关系，否则编译错误；

接口注意事项：①接口中定义的静态方法，只能通过接口来调用。

②通过实现类的对象，可以调用接口中定义的默认方法，如果实现类重写了此方法，则调用的是重写之后的方法。

动作识别笔记

2022-03-24

基于图像的动作识别较为成熟，针对视频的动作识别主要有以下几个难题：

①数据集中不确定因素较多，光照、环境变化、物体形变(人物交互)、目标遮挡(人人交互)都会影响模型性能；

②动作边界的不确定性：动作快、时间短的视频难以在时序上精准定位边界。

③对于目前主流的对细粒度视频动作的识别中，动作差异性较小，细微差距难以分辨带来较大挑战。

传统(将手工提取到的特征编码成高维的特征向量，然后利用特征向量训练分类器以进行动作识别，较高的时空复杂度，严重影响到算法的性能)：基于人体关节点、基于时空兴趣点和基于密集轨迹的特征提取

基于深度学习：2D CNN；3D CNN；时空分解网络；

2D CNN：（第一阶段）针对单流的 2D CNN 无法建模时间信息的问题，Simonyan提出了双流网络，空间流网络以单帧 RGB 图像作为输入，用于建模外观特征；时间流网络以堆叠的光流图像作为输入，用于建模运动特征。训练时，空间流网络和时间流网络单独地训练。测试时，通过平均所有采样的视频帧汇总两个流的softmax得分，从而得到视频级预测结果。

光流仅表示相邻帧之间的运动信息，因此双流网络对时间上下文的访问十分有限，Ng 等[12] 提出采用长短期记忆网络（long short-term memory，LSTM），LSTM 能够更加有效地表达视频帧在时间序列上的依赖关系，从而实现对长时间序列进行建模。然而，其对于视频帧之间底层的时间信息建模是不够的，会造成时序信息的丢失，并且 LSTM 的引入会导致大量的计算开销，不利于网络后期的优化。

因此，Wang等[14]提出了时域分段网（temporal segment networks，TSN）

以上都需要预计算光流，光流的预计算和存储都十分昂贵，并且光流会导致设备延迟，限制了性能。

（第二阶段）避免光流的提取：两种思路，一种思路是从光流算法的原理出发，通过设计卷积模块去隐式地模拟光流的提取过程，从而达到建模时间特征的目的；另一种思路是从光流算法的原理出发，通过设计卷积模块去隐式地模拟光流的提取过程，从而达到建模时间特征的目的；P4

3D CNN： Learning spatiotemporal features with 3D convolutional networks（里程碑）

(新模型的探索)网络架构探索方面：拓展出Res3D：通过改变各卷积层滤波器的个数来保持网络架构的参数一致，从而探讨了输入帧的采样频率、空间分辨率以及卷积的类型对模型性能的影响参数量较C3D较少，性能得以提高；T-C3D(时域三维卷积网络)：T-C3D 引入了 TSN 的视频级监督方法，同时确保各片段的 3D CNN 共享权值，既有利于得到全局的时空特征，又不产生额外的参数。

(现有模型的一系列精简操作以及优化)优化现有模型方面(涉及较为详细的优化操作)：

基于时空分解网络：通过将时空特征进行分开建模，以达到减少参数和易于优化的目的。

神经网络知识点

2022-03-09

前向传播过程：输入层给定X，经过一次计算后得到z，再经过激活函数后得到a[1]，再将a[1]作为第二次的输入进行迭代计算，n次之后得到a[n]=y_hat(预测值)，即为一次正向传播过程，也可将x理解为a[0]。

反向传播过程：根据y值不断地向前求导得出dz、dw的过程 。

激活函数的选择一般选择tanh函数，关于原点对称，数据平均值更接近于0

sigmoid函数一般只用在二分类问题。

不同层的激活函数可以不一样。

Relu函数a=max(0,z) 通常效果比tanh和sigmoid函数好，因为其不存在斜率为0的地方/也可用a=max(0.01z,z)

logistic回归，使用激活函数将第一躺得到的值作为第二趟的输入。

进行一趟深度学习的过程 P41搭建神经网络块。

超参数：深度学习中控制实际参数的参数，如学习率，隐藏层层数，隐藏层的单元数，循环次数，激活函数，这些参数决定着w,b的变化，所以称为超参数。

偏差和方差：过拟合和欠拟合都有可能造成高偏差和高方差。

偏差较高：调整神经网络，更多的隐藏层或者隐藏单元格，设置复杂的神经网络来尝试调整，偏差降低后再调整方差，常见的调整方差高的办法是更大的数据集，但偏差过高时增大数据集一般作用不大。

正则化：有助于缓解过拟合； L2正则化，通过使用lambd参数来调整w。

增大lambd 导致w[l]降低,最后导致z 减小，使得激活函数趋于线性，缓解过拟合，p50-51为推导过程。

dropout正则化：根据概率（各隐藏层节点每趟被保留的概率会变，每层的keep-prob都可能不同）每趟随机丢失一些节点，缓解过拟合。

inverted dropout：通过除去一些结点得到的每层的a值后，将其除以结点丢失的概率，确保在数量上弥补丢弃的部分数据，使计算下一趟a[l]时预期的a[l-1]不变，常见此法性能较优。

也可使用early stopping 来提早停止训练神经网络，通过找到w的较小值和较大值，再找出一个中间值来判断是否停止。 缓解过拟合。

提高训练速度

归一化输入：先零均值化，将节点调整至原点周围；再归一化方差，使其对应于两轴的布局均匀。（以二维为例）

梯度消失和梯度爆炸：使用线性激活函数时，w值指数型变化，w大于1，w^l极大梯度爆炸，w小于1梯度消失

梯度检验通常使用双边误差 即求导点的左右端定义。

k折交叉验证，将数据集k等分，每次取其中一个作为验证集，其他的作为训练集，一共进行k次，得出k次的验证概率，最后取平均值则为最终的验证概率。常用k取5或者10。

随机梯度下降（样本大小为1时），选择合适大小的样本进行梯度下降，样本数量太小则会失去向量化带来的加速，受噪声影响较大进而导致损失函数一直在均值之间波动不会停留，因为梯队下降无法收敛。数量太大一次性执行较大样本比较耗费时间，mini-batch size尽量为中间某个合适值。

指数加权平均，优点：指数计算在内存中只占用单行数字，如果需要获得刚精确得平均值，按正常的计算会更准确，但是保存过去的数值以及保存所求的最近几天的数值之和会比较占用内存

学习率衰退：α1=[1/(1+decay-rate(下降率)*epoch-n)]α0,一次遍历衰退一次，一次为一代，也可为其他公式使学习率α随时间变化下降 ，目的是在损失函数接近均值时，降低学习率以减小在均值附近的波动。

多个输入通道，每个通道都有一个卷积核，结果是所有通道卷积结果的和=多个输入通道得到单个输出通道

可以有多个三维卷积核，每个核生成一个输出通道，得到多输出通道
